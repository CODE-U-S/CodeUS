### 커밋 메시지

커밋 메시지는 다음과 같은 형식을 사용해야 합니다:

<타입>(<범위>): <설명>

### 타입

- **feat**: 새로운 기능 추가
- **fix**: 버그 수정
- **docs**: 문서 변경
- **style**: 코드 포맷팅
- **refactor**: 리팩토링
- **test**: 테스트 추가/수정
- **chore**: 기타 변경 사항

### 범위

- **auth**: 인증 관련
- **ui**: 사용자 인터페이스
- **api**: API
- **root**: 루트 디렉토리에서의 작업

커밋 메시지는 가능한 한 명확하고 구체적으로 작성해 주세요.
